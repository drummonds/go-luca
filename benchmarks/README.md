# Benchmark Document Assembly

Each benchmark directory produces a single HTML page via `luca-docs`.

## File convention

| File | Author | Description |
|------|--------|-------------|
| `purpose.md` | Human | Test objectives and questions to answer (protected, not auto-edited) |
| `analysis.md` | AI | Findings and recommendations |
| `summary.md` | AI | Executive summary (optional) |
| `results.md` | Benchmark program | Complete data-driven report with tables (generated by `cmd/bench-*`) |

## Assembly rules

`luca-docs` applies this logic per subdirectory:

1. If `results.md` exists, render it directly (it is the complete report).
2. Otherwise, merge in order: heading from dir name, then `purpose.md`, `analysis.md`, `summary.md`.

Output: `docs/benchmarks/<name>.html`

## Adding a benchmark

1. Create `benchmarks/<name>/purpose.md` describing what you want to measure.
2. Optionally write a standalone benchmark program in `cmd/bench-<name>/`.
3. Run the benchmark — it writes `results.md` into the benchmark directory.
4. If no program exists, add `analysis.md` and/or `summary.md` manually.
5. Run `task docs:build` to regenerate HTML.
